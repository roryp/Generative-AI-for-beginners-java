## ಅವಲೋಕನ

ಈ ಟ್ಯುಟೋರಿಯಲ್ ಜಾವಾ ಮತ್ತು GitHub ಮಾದರಿಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಕೋರ್ ಜನರೇಟಿವ್ AI ತಂತ್ರಗಳ ಕೈಯಲ್ಲಿ ಅನುಭವವನ್ನು ಒದಗಿಸುತ್ತದೆ. ನೀವು ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾದರಿಗಳ (LLMs) ಜೊತೆ ಸಂವಹನ ಮಾಡುವುದು, ಫಂಕ್ಷನ್ ಕಾಲಿಂಗ್ ಅನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು, ರಿಟ್ರೀವಲ್-ಆಗ್ಮೆಂಟೆಡ್ ಜನರೇಶನ್ (RAG) ಬಳಸುವುದು, ಮತ್ತು ಜವಾಬ್ದಾರಿಯುತ AI ಅಭ್ಯಾಸಗಳನ್ನು ಅನ್ವಯಿಸುವುದು ಕಲಿಯುತ್ತೀರಿ.

## ಪ್ರಾರಂಭಿಸುತ್ತಿರುವುದು

### ಹಂತ 1: ನಿಮ್ಮ ಪರಿಸರ ವ್ಯತ್ಯಾಸವನ್ನು ಹೊಂದಿಸಿ

ಮೊದಲು, ನಿಮ್ಮ GitHub ಟೋಕನ್ ಅನ್ನು ಪರಿಸರ ವ್ಯತ್ಯಾಸವಾಗಿ ಹೊಂದಿಸಬೇಕು. ಈ ಟೋಕನ್ ನಿಮಗೆ GitHub ಮಾದರಿಗಳನ್ನು ಉಚಿತವಾಗಿ ಪ್ರವೇಶಿಸಲು ಅನುಮತಿಸುತ್ತದೆ.

**Windows (ಕಮಾಂಡ್ ಪ್ರಾಂಪ್ಟ್):**
```cmd
set GITHUB_TOKEN=your_github_token_here
```

**Windows (ಪವರ್‌ಶೆಲ್):**
```powershell
$env:GITHUB_TOKEN="your_github_token_here"
```

**Linux/macOS:**
```bash
export GITHUB_TOKEN=your_github_token_here
```

## ಮಾದರಿ ಆಯ್ಕೆ ಮಾರ್ಗದರ್ಶಿ

ಈ ಉದಾಹರಣೆಗಳು ತಮ್ಮ ವಿಶೇಷ ಬಳಕೆ ಪ್ರಕರಣಗಳಿಗೆ ಆಪ್ಟಿಮೈಸ್ ಮಾಡಲಾದ ವಿಭಿನ್ನ ಮಾದರಿಗಳನ್ನು ಬಳಸುತ್ತವೆ:

**GPT-4.1-nano** (ಪೂರ್ಣಗೊಳಿಕೆಗಳ ಉದಾಹರಣೆ):
- ಅತೀ ವೇಗ ಮತ್ತು ಅತೀ ಕಡಿಮೆ ವೆಚ್ಚ
- ಮೂಲ ಪಠ್ಯ ಪೂರ್ಣಗೊಳಿಕೆ ಮತ್ತು ಚಾಟ್‌ಗೆ ಪರಿಪೂರ್ಣ
- ಮೂಲ LLM ಸಂವಹನ ಮಾದರಿಗಳನ್ನು ಕಲಿಯಲು ಆದರ್ಶ

**GPT-4o-mini** (ಫಂಕ್ಷನ್, RAG, ಮತ್ತು ಜವಾಬ್ದಾರಿಯುತ AI ಉದಾಹರಣೆಗಳು):
- ಸಣ್ಣ ಆದರೆ ಸಂಪೂರ್ಣ ವೈಶಿಷ್ಟ್ಯಗಳ "ಒಮ್ನಿ ವರ್ಕ್‌ಹಾರ್ಸ್" ಮಾದರಿ
- ವಿವಿಧ ವೆಂಡರ್‌ಗಳಲ್ಲಿ ಮುಂದುವರಿದ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನಂಬಿಕೆಯಿಂದ ಬೆಂಬಲಿಸುತ್ತದೆ:
  - ದೃಶ್ಯ ಪ್ರಕ್ರಿಯೆ
  - JSON/ಸಂರಚಿತ ಔಟ್‌ಪುಟ್‌ಗಳು  
  - ಟೂಲ್/ಫಂಕ್ಷನ್ ಕಾಲಿಂಗ್
- ನ್ಯಾನೋ ಮಾದರಿಗಿಂತ ಹೆಚ್ಚು ಸಾಮರ್ಥ್ಯ, ಉದಾಹರಣೆಗಳು ನಿರಂತರವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸಲು ಖಚಿತಪಡಿಸುತ್ತದೆ

> **ಇದು ಏಕೆ ಮುಖ್ಯವಾಗಿದೆ**: "ನ್ಯಾನೋ" ಮಾದರಿಗಳು ವೇಗ ಮತ್ತು ವೆಚ್ಚಕ್ಕೆ ಉತ್ತಮವಾದವು, ಆದರೆ "ಮಿನಿ" ಮಾದರಿಗಳು ಫಂಕ್ಷನ್ ಕಾಲಿಂಗ್ ಮುಂತಾದ ಮುಂದುವರಿದ ವೈಶಿಷ್ಟ್ಯಗಳಿಗೆ ನಂಬಿಕಾರ್ಹ ಪ್ರವೇಶವನ್ನು ಬೇಕಾದಾಗ ಸುರಕ್ಷಿತ ಆಯ್ಕೆಯಾಗಿದೆ.

## ಟ್ಯುಟೋರಿಯಲ್ 1: LLM ಪೂರ್ಣಗೊಳಿಕೆಗಳು ಮತ್ತು ಚಾಟ್

**ಫೈಲ್:** `src/main/java/com/example/genai/techniques/completions/LLMCompletionsApp.java`

### ಈ ಉದಾಹರಣೆ ಏನು ಕಲಿಸುತ್ತದೆ

ಈ ಉದಾಹರಣೆ OpenAI API ಮೂಲಕ ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾದರಿಗಳ (LLM) ಸಂವಹನದ ಮೂಲ ತಂತ್ರಗಳನ್ನು ತೋರಿಸುತ್ತದೆ, ಇದರಲ್ಲಿ GitHub ಮಾದರಿಗಳೊಂದಿಗೆ ಕ್ಲೈಂಟ್ ಪ್ರಾರಂಭ, ಸಿಸ್ಟಮ್ ಮತ್ತು ಬಳಕೆದಾರ ಪ್ರಾಂಪ್ಟ್‌ಗಳ ಸಂದೇಶ ರಚನೆ ಮಾದರಿಗಳು, ಸಂದೇಶ ಇತಿಹಾಸ ಸಂಗ್ರಹಣೆಯ ಮೂಲಕ ಸಂಭಾಷಣಾ ಸ್ಥಿತಿಯ ನಿರ್ವಹಣೆ, ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆಯ ಉದ್ದ ಮತ್ತು ಸೃಜನಶೀಲತೆಯ ಮಟ್ಟಗಳನ್ನು ನಿಯಂತ್ರಿಸಲು ಪ್ಯಾರಾಮೀಟರ್ ಟ್ಯೂನಿಂಗ್ ಸೇರಿವೆ.

### ಮುಖ್ಯ ಕೋಡ್ ತತ್ವಗಳು

#### 1. ಕ್ಲೈಂಟ್ ಸೆಟಪ್
```java
// AI ಕ್ಲೈಂಟ್ ಅನ್ನು ರಚಿಸಿ
OpenAIClient client = new OpenAIClientBuilder()
    .endpoint("https://models.inference.ai.azure.com")
    .credential(new StaticTokenCredential(pat))
    .buildClient();
```

ಇದು ನಿಮ್ಮ ಟೋಕನ್ ಬಳಸಿ GitHub ಮಾದರಿಗಳೊಂದಿಗೆ ಸಂಪರ್ಕವನ್ನು ರಚಿಸುತ್ತದೆ.

#### 2. ಸರಳ ಪೂರ್ಣಗೊಳಿಕೆ
```java
List<ChatRequestMessage> messages = List.of(
    // ಸಿಸ್ಟಮ್ ಸಂದೇಶ AI ನ ವರ್ತನೆವನ್ನು ಹೊಂದಿಸುತ್ತದೆ
    new ChatRequestSystemMessage("You are a helpful Java expert."),
    // ಬಳಕೆದಾರ ಸಂದೇಶದಲ್ಲಿ ನಿಜವಾದ ಪ್ರಶ್ನೆ ಇದೆ
    new ChatRequestUserMessage("Explain Java streams briefly.")
);

ChatCompletionsOptions options = new ChatCompletionsOptions(messages)
    .setModel("gpt-4.1-nano")  // ಮೂಲ ಪೂರ್ಣಗೊಳಿಸುವಿಕೆಗಳಿಗೆ ವೇಗದ, ವೆಚ್ಚ-ಪ್ರಭಾವಿ ಮಾದರಿ
    .setMaxTokens(200)         // ಪ್ರತಿಸ್ಪಂದನೆಯ ಉದ್ದವನ್ನು ಮಿತಿಗೊಳಿಸಿ
    .setTemperature(0.7);      // ಸೃಜನಶೀಲತೆಯನ್ನು ನಿಯಂತ್ರಿಸಿ (0.0-1.0)
```

#### 3. ಸಂಭಾಷಣಾ ಮೆಮೊರಿ
```java
// AI ಯ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಸೇರಿಸಿ ಸಂಭಾಷಣೆಯ ಇತಿಹಾಸವನ್ನು ನಿರ್ವಹಿಸಲು
messages.add(new ChatRequestAssistantMessage(aiResponse));
messages.add(new ChatRequestUserMessage("Follow-up question"));
```

AI ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ಮಾತ್ರ ನೀವು ಅವುಗಳನ್ನು ಮುಂದಿನ ವಿನಂತಿಗಳಲ್ಲಿ ಸೇರಿಸಿದರೆ ನೆನಪಿಡುತ್ತದೆ.

### ಉದಾಹರಣೆಯನ್ನು ಚಲಾಯಿಸಿ
```bash
mvn compile exec:java -Dexec.mainClass="com.example.genai.techniques.completions.LLMCompletionsApp"
```

### ನೀವು ಇದನ್ನು ಚಲಾಯಿಸಿದಾಗ ಏನಾಗುತ್ತದೆ

1. **ಸರಳ ಪೂರ್ಣಗೊಳಿಕೆ**: AI ಜಾವಾ ಪ್ರಶ್ನೆಗೆ ಸಿಸ್ಟಮ್ ಪ್ರಾಂಪ್ಟ್ ಮಾರ್ಗದರ್ಶನದೊಂದಿಗೆ ಉತ್ತರಿಸುತ್ತದೆ
2. **ಮಲ್ಟಿ-ಟರ್ನ್ ಚಾಟ್**: AI ಹಲವಾರು ಪ್ರಶ್ನೆಗಳಾದ್ಯಂತ ಪ್ರಾಸಂಗಿಕತೆಯನ್ನು ಕಾಪಾಡುತ್ತದೆ
3. **ಇಂಟರಾಕ್ಟಿವ್ ಚಾಟ್**: ನೀವು AI ಜೊತೆ ನಿಜವಾದ ಸಂಭಾಷಣೆಯನ್ನು ನಡೆಸಬಹುದು
