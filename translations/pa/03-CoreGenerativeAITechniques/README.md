## ਮਾਡਲ ਚੋਣ ਮਾਰਗਦਰਸ਼ਨ

ਇਹ ਉਦਾਹਰਣ ਵੱਖ-ਵੱਖ ਮਾਡਲਾਂ ਦੀ ਵਰਤੋਂ ਕਰਦੀਆਂ ਹਨ ਜੋ ਆਪਣੇ ਵਿਸ਼ੇਸ਼ ਵਰਤੋਂ ਮਾਮਲਿਆਂ ਲਈ ਵਿਸ਼ੇਸ਼ ਅਨੁਕੂਲ ਹਨ:

**GPT-4.1-nano** (ਪੂਰਨਤਾ ਉਦਾਹਰਣ):
- ਅਤੀ ਤੇਜ਼ ਅਤੇ ਬਹੁਤ ਹੀ ਸਸਤਾ
- ਬੁਨਿਆਦੀ ਟੈਕਸਟ ਪੂਰਨਤਾ ਅਤੇ ਚੈਟ ਲਈ ਉੱਤਮ
- ਮੂਲ LLM ਇੰਟਰੈਕਸ਼ਨ ਪੈਟਰਨ ਸਿੱਖਣ ਲਈ ਪਰਫੈਕਟ

**GPT-4o-mini** (ਫੰਕਸ਼ਨ, RAG, ਅਤੇ ਜ਼ਿੰਮੇਵਾਰ AI ਉਦਾਹਰਣਾਂ):
- ਛੋਟਾ ਪਰ ਪੂਰੀ ਤਰ੍ਹਾਂ ਸੂਚਿਤ "ਸਾਰਥਕ ਕੰਮ ਦਾ ਘੋੜਾ" ਮਾਡਲ
- ਵਿਕਰੇਤਾਵਾਂ ਵਿੱਚ ਵਿਕਸਤ ਸਮਰੱਥਾਵਾਂ ਲਈ ਭਰੋਸੇਯੋਗ ਸਹਿਯੋਗ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ:
  - ਦ੍ਰਿਸ਼ ਪ੍ਰੋਸੈਸਿੰਗ
  - JSON/ਸੰਰਚਿਤ ਆਉਟਪੁੱਟਾਂ  
  - ਟੂਲ/ਫੰਕਸ਼ਨ ਕਾਲਿੰਗ
- ਨੈਨਾ ਦੇ ਨਾਲੋਂ ਵੱਧ ਸਮਰੱਥਾਵਾਂ, ਜਿਸ ਨਾਲ ਉਦਾਹਰਣ ਸਥਿਰ ਤੌਰ 'ਤੇ ਕੰਮ ਕਰਦੇ ਹਨ

> **ਇਹ ਮਹੱਤਵਪੂਰਨ ਕਿਉਂ ਹੈ**: ਜਿੱਥੇ "ਨੈਨਾ" ਮਾਡਲ ਤੇਜ਼ੀ ਅਤੇ ਲਾਗਤ ਲਈ ਬਿਹਤਰੀਨ ਹਨ, "ਮੀਨੀ" ਮਾਡਲ ਵਿਕਸਤ ਫੰਕਸ਼ਨ ਕਾਲਿੰਗ ਵਰਗੀਆਂ ਅਡਵਾਂਸ ਸਹੂਲਤਾਂ ਲਈ ਭਰੋਸੇਯੋਗ ਚੋਣ ਹਨ, ਜੋ ਕਿ ਸਾਰੇ ਹੋਸਟਿੰਗ ਪ੍ਰਦਾਤਾ ਨੈਨਾ ਵਰਜਨਾਂ ਵਿੱਚ ਪੂਰੀ ਤਰ੍ਹਾਂ ਉਪਲਬਧ ਨਹੀਂ ਕਰਵਾਉਂਦੇ।

## ਟਿਊਟੋਰਿਯਲ 1: LLM ਪੂਰਨਤਾ ਅਤੇ ਚੈਟ

**ਫਾਈਲ:** `src/main/java/com/example/genai/techniques/completions/LLMCompletionsApp.java`

### ਇਹ ਉਦਾਹਰਣ ਕੀ ਸਿੱਖਾਉਂਦਾ ਹੈ

ਇਹ ਉਦਾਹਰਣ ਵੱਡੇ ਭਾਸ਼ਾ ਮਾਡਲ (LLM) ਇੰਟਰੈਕਸ਼ਨ ਦੀ ਕੋਰ ਮਕੈਨੀਕਿਜ਼ ਨੂੰ ਦਰਸਾਂਦਾ ਹੈ ਜੋ OpenAI API ਰਾਹੀਂ ਹੈ, ਜਿਸ ਵਿੱਚ GitHub ਮਾਡਲਾਂ ਨਾਲ ਕਲਾਇੰਟ ਇਨਿਸ਼ੀਅਲਾਈਜੇਸ਼ਨ, ਸਿਸਟਮ ਅਤੇ ਯੂਜ਼ਰ ਪ੍ਰਾੰਪਟਾਂ ਲਈ ਸੁਨੇਹੇ ਦੀ ਬਣਤਰ ਦੇ ਪੈਟਰਨ, ਗੱਲਬਾਤ ਦੀ ਹਾਲਤ ਨੂੰ ਸੁਨੇਹਾ ਇਤਿਹਾਸ ਰਾਹੀਂ ਸੰਭਾਲਣਾ, ਅਤੇ ਜਵਾਬ ਦੀ ਲੰਬਾਈ ਅਤੇ ਰਚਨਾਤਮਕਤਾ ਦੀ ਸਥਿਤੀ ਲਈ ਪੈਰਾਮੀਟਰ ਟਿਊਨਿੰਗ ਸ਼ਾਮਿਲ ਹੈ।

### ਮੁੱਖ ਕੋਡ ਧਾਰਨਾਵਾਂ

#### 1. ਕਲਾਇੰਟ ਸੈੱਟਅੱਪ
```java
// ਏਆਈ ਕਲਾਇੰਟ ਬਣਾਓ
OpenAIClient client = new OpenAIClientBuilder()
    .endpoint("https://models.inference.ai.azure.com")
    .credential(new StaticTokenCredential(pat))
    .buildClient();
```

ਇਹ ਤੁਹਾਡੇ ਟੋਕਨ ਦੀ ਵਰਤੋਂ ਕਰਕੇ GitHub ਮਾਡਲਾਂ ਨਾਲ ਕਨੈਕਸ਼ਨ ਬਣਾਉਂਦਾ ਹੈ।

#### 2. ਸਧਾਰਣ ਪੂਰਨਤਾ
```java
List<ChatRequestMessage> messages = List.of(
    // ਸਿਸਟਮ ਸੁਨੇਹਾ AI ਵਤੀਰੇ ਨੂੰ ਸੈਟ ਕਰਦਾ ਹੈ
    new ChatRequestSystemMessage("You are a helpful Java expert."),
    // ਯੂਜ਼ਰ ਸੁਨੇਹਾ ਅਸਲ ਸਵਾਲ ਸ਼ਾਮਲ ਕਰਦਾ ਹੈ
    new ChatRequestUserMessage("Explain Java streams briefly.")
);

ChatCompletionsOptions options = new ChatCompletionsOptions(messages)
    .setModel("gpt-4.1-nano")  // ਬੁਨਿਆਦੀ ਪੂਰਾ ਕਰਨ ਲਈ ਤੇਜ਼, ਲਾਗਤ-ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਮਾਡਲ
    .setMaxTokens(200)         // ਜਵਾਬ ਦੀ ਲੰਬਾਈ ਸੀਮਤ ਕਰੋ
    .setTemperature(0.7);      // ਰਚਨਾਤਮਕਤਾ ਨੂੰ ਨਿਯੰਤਰਿਤ ਕਰੋ (0.0-1.0)
```

#### 3. ਗੱਲਬਾਤੀ ਯਾਦ
```java
// ਗੱਲਬਾਤ ਦੇ ਇਤਿਹਾਸ ਨੂੰ ਬਣਾਈ ਰੱਖਣ ਲਈ AI ਦੀ ਪ੍ਰਤੀਕ੍ਰਿਆ ਸ਼ਾਮਲ ਕਰੋ
messages.add(new ChatRequestAssistantMessage(aiResponse));
messages.add(new ChatRequestUserMessage("Follow-up question"));
```

ਗੱਲਬਾਤ ਪਿਛਲੇ ਸੁਨੇਹਿਆਂ ਨੂੰ ਸਿਰਫ਼ ਤਦ ਯਾਦ ਰੱਖਦੀ ਹੈ ਜਦ ਤੁਹਾਨੂੰ ਉਹ ਅਗਲੇ ਬਾਰੇ ਵਿੱਚ ਸ਼ਾਮਿਲ ਕਰੋ।

## ਟਿਊਟੋਰਿਯਲ 2: ਫੰਕਸ਼ਨ ਕਾਲਿੰਗ

**ਫਾਈਲ:** `src/main/java/com/example/genai/techniques/functions/FunctionsApp.java`

### ਇਹ ਉਦਾਹਰਣ ਕੀ ਸਿੱਖਾਉਂਦਾ ਹੈ

ਫੰਕਸ਼ਨ ਕਾਲਿੰਗ AI ਮਾਡਲਾਂ ਨੂੰ ਬਾਹਰੀ ਟੂਲਾਂ ਅਤੇ APIਜ਼ ਦੇ ਨਿਰੁਪਨੇ ਅਨੁਸਾਰ ਫੰਕਸ਼ਨਾਂ ਨੂੰ ਕਾਲ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ, ਜਿੱਥੇ ਮਾਡਲ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਅਨੁਰੋਧਾਂ ਦਾ ਵਿਸ਼ਲੇਸ਼ਣ ਕਰਦਾ ਹੈ, JSON ਸਕੀਮਾ ਵਿਆਖਿਆਵਾਂ ਦਾ ਇਸਤੇਮਾਲ ਕਰ ਕੇ ਲੋੜੀਂਦੇ ਪੈਰਾਮੀਟਰਾਂ ਨਾਲ ਸਹੀ ਫੰਕਸ਼ਨਾਂ ਨੂੰ ਚੁਣਦਾ ਹੈ, ਅਤੇ ਮਿਲੇ ਹੋਏ ਨਤੀਜਿਆਂ ਨੂੰ ਪ੍ਰਸੰਗਿਕ ਜਵਾਬਾਂ ਵਿਚ ਪਰਿਵਰਤਿਤ ਕਰਦਾ ਹੈ, ਜਦ ਔਗਤਕਾਰ ਸੁਰੱਖਿਆ ਅਤੇ ਭਰੋਸੇਯੋਗਤਾ ਦੇ ਲਈ ਫੰਕਸ਼ਨ ਲਾਗੂਕਰਨ ਤੇ ਕਾਬੂ ਰੱਖਦਾ ਹੈ।

> **ਨੋਟ**: ਇਹ ਉਦਾਹਰਣ `gpt-4o-mini` ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ ਕਿਉਂਕਿ ਫੰਕਸ਼ਨ ਕਾਲਿੰਗ ਲਈ ਭਰੋਸੇਯੋਗ ਟੂਲ ਕਾਲਿੰਗ ਸਮਰੱਥਾਵਾਂ ਚਾਹੀਦੀਆਂ ਹਨ ਜੋ ਸਾਰੇ ਹੋਸਟਿੰਗ ਪਲੇਟਫਾਰਮਾਂ 'ਤੇ ਨੈਨਾ ਮਾਡਲਾਂ ਵਿੱਚ ਪੂਰੀ ਤਰ੍ਹਾਂ ਉਪਲਬਧ ਨਹੀਂ ਹੁੰਦੀਆਂ।

### ਮੁੱਖ ਕੋਡ ਧਾਰਨਾਵਾਂ

#### 1. ਫੰਕਸ਼ਨ ਪਰਿਭਾਸ਼ਾ
```java
ChatCompletionsFunctionToolDefinitionFunction weatherFunction = 
    new ChatCompletionsFunctionToolDefinitionFunction("get_weather");
weatherFunction.setDescription("Get current weather information for a city");

// JSON ਸਕੀਮਾ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਪੈਰਾਮੀਟਰ ਪਰਿਭਾਸ਼ਿਤ ਕਰੋ
weatherFunction.setParameters(BinaryData.fromString("""
    {
        "type": "object",
        "properties": {
            "city": {
                "type": "string",
                "description": "The city name"
            }
        },
        "required": ["city"]
    }
    """));
```

ਇਹ AI ਨੂੰ ਦੱਸਦਾ ਹੈ ਕਿ ਕਿਹੜੇ ਫੰਕਸ਼ਨ ਉਪਲਬਧ ਹਨ ਅਤੇ ਕਿਵੇਂ ਵਰਤੇ ਜਾਣ।

## ਟਿਊਟੋਰਿਯਲ 3: RAG (ਰੀਟਰੀਵਲ-ਆਗਮੈਂਟਡ ਜਨਰੇਸ਼ਨ)

### ਇਹ ਉਦਾਹਰਣ ਕੀ ਸਿੱਖਾਉਂਦਾ ਹੈ

ਰੀਟਰੀਵਲ-ਆਗਮੈਂਟਡ ਜਨਰੇਸ਼ਨ (RAG) ਜਾਣਕਾਰੀ ਪਹੁੰਚਾਈ ਅਤੇ ਭਾਸ਼ਾ ਜਨਰੇਸ਼ਨ ਨੂੰ ਜੋੜਦਾ ਹੈ ਜਿਸ ਨਾਲ ਬਾਹਰੀ ਦਸਤਾਵੇਜ਼ ਸੰਦਰਭ AI ਪ੍ਰਾੰਪਟਾਂ ਵਿੱਚ ਜੋੜ ਦਿੱਤਾ ਜਾਂਦਾ ਹੈ, ਤਾਂ ਜੋ ਮਾਡਲ ਨਿਰਧਾਰਿਤ ਹਵਾਲੇ ਵਾਲੇ ਸਰੋਤਾਂ 'ਤੇ ਆਧਾਰਿਤ ਸਹੀ ਜਵਾਬ ਦੇ ਸਕਣ ਜਦਕਿ ਸਮਭਵ ਤੌਰ ਤੇ ਪੁਰਾਣੇ ਜਾਂ ਗਲਤ ਪ੍ਰਸ਼ਿਸ਼ਣ ਡੇਟਾ ਤੋਂ ਬਚਾਵ ਕੀਤਾ ਜਾਵੇ, ਅਤੇ ਯੂਜ਼ਰ ਪ੍ਰਸ਼ਨਾਂ ਅਤੇ ਪ੍ਰਮਾਣਿਤ ਜਾਣਕਾਰੀ ਸਰੋਤਾਂ ਵਿਚਕਾਰ ਸਪਸ਼ਟ ਸੀਮਾਵਾਂ ਨੂੰ ਬਣਾਈ ਰੱਖਣ ਲਈ ਸਟ੍ਰੈਟਿਜਿਕ ਪ੍ਰਾੰਪਟ ਇੰਜੀਨੀਅਰਿੰਗ ਵਰਤੀ ਜਾਂਦੀ ਹੈ।

> **ਨੋਟ**: ਇਹ ਉਦਾਹਰਣ `gpt-4o-mini` ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ ਤਾਂ ਜੋ ਸੰਰਚਿਤ ਪ੍ਰਾੰਪਟਾਂ ਦੀ ਭਰੋਸੇਯੋਗ ਪ੍ਰੋਸੈਸਿੰਗ ਅਤੇ ਦਸਤਾਵੇਜ਼ ਸੰਦਰਭ ਦੀ ਸਮਰੱਥਾ ਨੂੰ ਯਕੀਨੀ ਬਨਾਇਆ ਜਾ ਸਕੇ ਜੋ ਪ੍ਰਭਾਵਸ਼ਾਲੀ RAG ਲਾਗੂਕਰਨ ਲਈ ਬਹੁਤ ਜਰੂਰੀ ਹੈ।

## ਟਿਊਟੋਰਿਯਲ 4: ਜ਼ਿੰਮੇਵਾਰ AI

### ਇਹ ਉਦਾਹਰਣ ਕੀ ਸਿੱਖਾਉਂਦਾ ਹੈ

ਜ਼ਿੰਮੇਵਾਰ AI ਉਦਾਹਰਣ AI ਐਪਲੀਕੇਸ਼ਨਾਂ ਵਿੱਚ ਸੁਰੱਖਿਆ ਉਪਾਇ ਲਾਗੂ ਕਰਨ ਦੇ ਮਹੱਤਵ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ। ਇਹ ਦਰਸਾਉਂਦਾ ਹੈ ਕਿ ਆਧੁਨਿਕ AI ਸੁਰੱਖਿਆ ਪ੍ਰਣਾਲੀਆਂ ਦੋ ਮੁੱਖ ਤਰੀਕਿਆਂ ਨਾਲ ਕੰਮ ਕਰਦੀਆਂ ਹਨ: ਹਾਰਡ ਬਲਾਕਸ (ਸੁਰੱਖਿਆ ਫਿਲਟਰਾਂ ਤੋਂ HTTP 400 ਗਲਤੀਆਂ) ਅਤੇ ਸਾਫਟ ਰਿਫਿਜ਼ਲ (ਮਾਡਲ ਵੱਲੋਂ ਨਿਮਰ "ਮੈਂ ਇਸ ਵਿੱਚ ਮਦਦ ਨਹੀਂ ਕਰ ਸਕਦਾ" ਦੇ ਜਵਾਬ)। ਇਹ ਉਦਾਹਰਣ ਵੇਖਾਉਂਦਾ ਹੈ ਕਿ ਉਤਪਾਦਨ AI ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਸਹੀ ਛੋਟੇ ਕੇਸ ਹੈਂਡਲਿੰਗ, ਇਨਕਾਰ ਪਤਾ ਲਗਾਉਣਾ, ਯੂਜ਼ਰ ਫੀਡਬੈਕ ਮਕੈਨਿਜ਼ਮ, ਅਤੇ ਬੈਕਅੱਪ ਜਵਾਬ ਰਣਨੀਤੀਆਂ ਨਾਲ ਸਮੱਗਰੀ ਨੀਤੀ ਉਲੰਘਣਾਂ ਨੂੰ ਕਿਵੇਂ ਨਰਮਾਈ ਨਾਲ ਸੰਭਾਲਣਾ ਚਾਹੀਦਾ ਹੈ।

> **ਨੋਟ**: ਇਹ ਉਦਾਹਰਣ `gpt-4o-mini` ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ ਕਿਉਂਕਿ ਇਹ ਵੱਖ-ਵੱਖ ਕਿਸਮਾਂ ਦੀ ਸੰਭਵ ਤੌਰ ਤੇ ਹਾਨਿਕਾਰਕ ਸਮੱਗਰੀ ਵਿੱਚ ਜ਼ਿਆਦਾ ਸਥਿਰ ਅਤੇ ਭਰੋਸੇਯੋਗ ਸੁਰੱਖਿਆ ਜਵਾਬ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ, ਇਹ ਯਕੀਨੀ ਬਣਾਉਂਦਾ ਹੈ ਕਿ ਸੁਰੱਖਿਆ ਮਕੈਨਿਜ਼ਮ ਸਹੀ ਢੰਗ ਨਾਲ ਦਰਸਾਏ ਜਾਂ।

#### 2. ਇਨਕਾਰ ਪਤਾ ਲਗਾਉਣਾ
```java
private boolean isRefusalResponse(String response) {
    String lowerResponse = response.toLowerCase();
    String[] refusalPatterns = {
        "i can't assist with", "i cannot assist with",
        "sorry, i can't", "sorry, i cannot",
        "i'm unable to", "against my guidelines"
    };
    
    for (String pattern : refusalPatterns) {
        if (lowerResponse.contains(pattern)) {
            return true;
        }
    }
    return false;
}
```
